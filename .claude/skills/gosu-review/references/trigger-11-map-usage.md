# Trigger 11: 맵 운용

> 이 파일은 gosu-review SKILL의 트리거 11 레퍼런스다.
> 메인 SKILL.md의 워크플로우 3단계에서 이 트리거가 발동하면 읽는다.

## 발동 조건

- `delete(m, ...)` 호출이 있는 맵
- 또는 **장수명 맵**: 전역 변수, 구조체 필드, 이름에 `cache`/`registry`/`store`/`pool` 포함

## 검사 절차

1. 맵의 라이프타임 추정 (지역? 필드? 전역?)
2. 삭제 빈도/규모 (대량 삭제가 반복되는가)
3. 값 타입의 크기 (큰 값을 직접 저장하고 있는가)

## 룰

### Rule 29 — 맵 메모리 단조 증가
- 핵심 질문: 맵이 한번 커지면 메모리가 줄지 않는다는 점을 의식했는가?
- 사실: `delete`로 항목을 지워도 백킹 버킷은 유지 → 메모리 사용량 감소 X
- 대량 삭제가 빈번한 경우의 옵션:
  - **주기적 재생성**: 새 맵에 살아있는 항목만 복사 후 교체
  - **값을 포인터로 보관**: `map[K]*V` — 값 자체의 메모리는 GC 가능
  - **다른 자료구조 검토**: LRU 캐시, sync.Map (한정된 패턴)
- 검사 시 맵의 라이프타임과 삭제 빈도를 함께 본다

## 출력 시 주의사항

`delete` 호출이 있다고 무조건 위반은 아니다. **장수명 + 대량 삭제** 조합일 때만 실질 위험. 근거에 두 조건을 함께 언급할 것.
