# Trigger 04: 가드 절/중첩 깊이

> 이 파일은 gosu-review SKILL의 트리거 04 레퍼런스다.
> 메인 SKILL.md의 워크플로우 3단계에서 이 트리거가 발동하면 읽는다.

## 발동 조건

함수 내부에서 `if`/`for`/`switch`의 중첩 깊이가 3 이상

## 검사 절차

1. 함수 본문을 읽으며 들여쓰기 깊이가 3단 이상 들어가는 지점을 찾는다
2. 각 분기가 "예외 처리(에러, 조기 실패)"인지 "정상 흐름의 일부"인지 판별
3. 예외 처리 분기가 정상 흐름을 감싸고 있으면 가드 절 리팩토링 후보

## 룰

### Rule 2 — 중첩 최소화 (happy path 좌측 정렬)
- 핵심 질문: 정상 경로가 함수의 가장 바깥 레벨에 있는가?
- 안티패턴:
  ```go
  if cond1 {
      if cond2 {
          if cond3 {
              // happy path
          }
      }
  }
  ```
- 권장 (가드 절):
  ```go
  if !cond1 { return ... }
  if !cond2 { return ... }
  if !cond3 { return ... }
  // happy path
  ```
- 적용 우선순위: 에러 분기 → 즉시 반환, `else` 제거, 깊은 분기는 함수 추출 검토
