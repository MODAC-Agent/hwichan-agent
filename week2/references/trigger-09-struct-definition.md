# Trigger 09: 구조체 정의 (`type ... struct`)

> 이 파일은 gosu-review SKILL의 트리거 09 레퍼런스다.
> 메인 SKILL.md의 워크플로우 3단계에서 이 트리거가 발동하면 읽는다.

## 발동 조건

`type Name struct { ... }` 선언

## 검사 절차

1. 임베딩(이름 없는 필드)이 있으면 가시성 검토
2. 같은 패키지/외부 패키지의 Get/Set 메서드 존재 여부

## 룰

### Rule 4 — 게터/세터 기계적 적용
- 핵심 질문: 모든 필드에 대해 `GetX()`/`SetX()` 메서드를 일률적으로 만들었는가?
- Go 관례: 단순 접근은 필드 노출(또는 직접 export). `GetX` 접두사도 비관용적 (`X()`만)
- 정당한 경우: 검증/계산이 필요한 경우, 인터페이스 충족, 동시성 가드
- 단순 노출이면 필드를 그대로 쓸 것

### Rule 10 — 임베딩 가시성
- 핵심 질문: 임베딩으로 보일러플레이트를 줄였는데, 외부에 노출되면 안 되는 메서드/필드가 promotion되지 않았는가?
- 위험:
  ```go
  type Cache struct {
      sync.Mutex  // Lock/Unlock이 외부에서 호출 가능해짐
  }
  ```
- 가이드: 내부 구현 세부면 **명명된 필드**로 (`mu sync.Mutex`)
- 인터페이스 임베딩도 동일 검토
